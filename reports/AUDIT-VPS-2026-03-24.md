## 2. Critical Finding: Code Mismatch

The VPS runs the **official upstream `ghcr.io/gitroomhq/postiz-app:latest`** image, NOT the custom fork.

Our local fork (`/root/postiz-fork/` on VPS) contains custom code with `getAppCredentials()` for per-channel X credentials, but it is **never built or used**. The docker-compose still points to the official image.

**Impact:** Per-channel X API credentials stored in `additionalSettings` are completely ignored. Only the global `X_API_KEY` env var matters.

| `authenticate()` saves env vars to additionalSettings | ✅ applied today | ❌ not deployed |

**To deploy custom code:**
1. Commit local changes (`git commit`)
2. Push to `https://github.com/unthai/postiz-app` (`git push`)
3. GitHub Actions builds `ghcr.io/unthai/postiz-app:latest`
4. Update `/home/unthaicore/postiz/docker-compose.yml` image to `ghcr.io/unthai/postiz-app:latest` (note: from `unthai` not `gitroomhq`)
5. `docker compose up -d --force-recreate postiz`

Or build directly on VPS:
```bash
cd /root/postiz-fork
docker build -f Dockerfile.dev -t postiz-custom:latest .
# Then update docker-compose.yml image: postiz-custom:latest
```
